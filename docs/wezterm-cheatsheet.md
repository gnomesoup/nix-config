# WezTerm Cheat Sheet

Your WezTerm config is managed in `users/modules/wezterm.nix`.

The tab bar is rendered at the bottom of the window. The right status shows the active workspace and domain.

## Leader

The leader key is `Ctrl-a` with a 1000 ms timeout.

After pressing `Ctrl-a`, use one of the bindings below before the leader timeout expires.

## Panes

These bindings use the active `vimBindingKeyboardLayout` from `users/modules/vimBindingKeyboardLayout.nix`.
With the default `colemak-dh` layout, the direction keys are `m/n/e/i`.

| Keys | Description |
| --- | --- |
| `Ctrl-a v` | Split the current pane horizontally |
| `Ctrl-a s` | Split the current pane vertically |
| `Ctrl-m` | Focus pane left |
| `Ctrl-n` | Focus pane down |
| `Ctrl-e` | Focus pane up |
| `Ctrl-i` | Focus pane right |
| `Ctrl-a r` | Enter resize mode |

## Resize Mode

Press `Ctrl-a r` and then use the active layout's direction keys to resize the active pane.

| Keys | Description |
| --- | --- |
| `m` | Resize left |
| `n` | Resize down |
| `e` | Resize up |
| `i` | Resize right |
| `Escape` / `Enter` | Exit resize mode |

Resize mode also exits after 2500 ms of inactivity or when you press an unmapped key.

## Copy Mode

Copy mode uses the active layout's Vim-style movement keys. With the default `colemak-dh` layout:

| Keys | Description |
| --- | --- |
| `m` / `n` / `e` / `i` | Move left / down / up / right |
| `w` / `b` / `f` | Move forward word / backward word / to word end |
| `t` / `T` | Jump forward / backward to character |
| `j` / `J` | Jump forward / backward until character |
| `k` / `K` | Next / previous search match while searching |

## Selection Helpers

| Keys | Description |
| --- | --- |
| `Ctrl-Shift-s` | Activate pane select |
| `Ctrl-a s` | Activate quick select |

Quick Select uses the active `vimBindingKeyboardLayout`'s `flashLabels` alphabet from `users/modules/vimBindingKeyboardLayout.nix`.
For the default `colemak-dh` layout, that alphabet is:

```text
arstgmeioqwfpbjluyxcdvzkh
```

## Projects, Tabs, and Domains

| Keys | Description |
| --- | --- |
| `Ctrl-a p` | Create/open persisted recent projects first, then active tabs |
| `Ctrl-a Shift-p` | Manage saved projects: open, rename, or delete |
| `Ctrl-a d` | Open a fuzzy domain launcher; selected domains open in a new tab named after the host |
| `Ctrl-a Shift-d` | Detach the current remote domain |

The project tab action supports local and WSL panes. If the current pane is inside an unsaved git repository, `Ctrl-a p` includes a `New project tab` entry. It names the tab after the current git repository and opens a project layout with `nvim`, a bottom shell pane, and a `pi` side pane.
Projects opened from the `Ctrl-a p` recent-project list are persisted in `~/.local/state/wezterm/recent-projects.json` so they survive WezTerm restarts. The fuzzy launcher lists those projects in most-recently-used order and only falls back to non-project tabs afterward. Its candidate labels start with the project/tab name, followed by the shortened path and domain; only the name is colored so highlighted rows stay readable, with active projects in green and inactive projects in purple. The selector is kept to 12 rows so fuzzy matching feels closer to a Spacemacs project switcher.

Use `Ctrl-a Shift-p` to manage saved projects. Rename changes the persisted project alias and attempts to rename a matching active tab; delete only removes the saved project entry and leaves any active tab running.

On macOS, normal app launches use the default local domain instead of auto-connecting to `local_mux`. The `local_mux` domain is still available from `Ctrl-a d` or `wezterm connect local_mux` when you explicitly want mux-backed persistent sessions.

## Tab Bar

| Area | Meaning |
| --- | --- |
| Tab title | Tab index and active pane title |
| Right status | Active workspace and active domain |

## Notes

- The config is generated by home-manager; edit `users/modules/wezterm.nix` and rebuild with your normal home-manager workflow.
- Directional pane focus, resize mode, copy mode, search mode, and Quick Select labels are derived from `vimBindingKeyboardLayout`.
- The default layout is currently `colemak-dh`, configured in `users/modules/vimBindingKeyboardLayout.nix`.
- WezTerm's default keybindings remain enabled unless overridden by this config.
